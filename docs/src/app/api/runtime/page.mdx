# API Reference: Runtime

`Runtime` is the central struct that drives a single zero-native app. It owns the platform surface, the window and webview tables, the in-flight async bridge responses, and the frame state machine. One `Runtime` is created per process and lives until shutdown. See [App Model](/app-model) for the lifecycle.

## Construction

```zig
var runtime = zero_native.Runtime.init(options);
try runtime.run(app);
```

`init` populates the surface from `options.platform.surface()`. `run` blocks on platform events until `.app_shutdown` is observed and is the only place the platform event loop is entered.

## Threading

Every `Runtime` field is owned by the thread that called `init`. Reading or writing them from a foreign thread is undefined behavior. The only exception is `next_stream_id`, which is an atomic counter so concurrent channel allocations do not collide.

See also: [App Model](/app-model), [Bridge](/bridge), [Channel](/api/channel).
