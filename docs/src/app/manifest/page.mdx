# Manifest Reference

The zero-native manifest declares app identity, permissions, bridge policies, security rules, window layout, and frontend configuration. In zero-native projects it lives at `app.zon` and uses ZON syntax; the schema below is the same regardless of how the manifest is serialized.

## Example

```toml
id = "com.example.myapp"
name = "myapp"
display_name = "My App"
version = "1.2.3"
icons = ["assets/icon.icns", "assets/icon.ico"]
platforms = ["macos", "linux"]
permissions = ["window", "filesystem"]
capabilities = ["webview", "js_bridge", "native_module"]

[[bridge.commands]]
name = "native.ping"
permissions = ["filesystem"]
origins = ["zero://app"]

[[bridge.commands]]
name = "zero-native.window.create"
permissions = ["window"]
origins = ["zero://app"]

[security.navigation]
allowed_origins = ["zero://app", "http://127.0.0.1:5173"]

[security.navigation.external_links]
action = "open_system_browser"
allowed_urls = ["https://example.com/*"]

web_engine = "system"

[cef]
dir = "third_party/cef/macos"
auto_install = false

[frontend]
dist = "frontend/dist"
entry = "index.html"
spa_fallback = true

[frontend.dev]
url = "http://127.0.0.1:5173/"
command = ["npm", "--prefix", "frontend", "run", "dev"]
ready_path = "/"
timeout_ms = 30000

[[windows]]
label = "main"
title = "My App"
width = 720
height = 480
x = 100
y = 100
restore_state = true
```

## Lifetime

The manifest parser allocates strings, arrays, and nested objects from the supplied allocator and returns them in a `Metadata` value. The caller owns the result and must free it with `Metadata.deinit(allocator)` to avoid leaks.

```zig
const metadata = try tooling.manifest.readMetadata(allocator, io, "app.zon");
defer metadata.deinit(allocator);
```
