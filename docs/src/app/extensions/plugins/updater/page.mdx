# Plugin: Updater

Auto-updater plugin. Parses update manifests, compares versions, and reports whether an update is available. `download` and `install` are stubs pending a real HTTP update server.

## Commands

| Command | Payload format | Description |
|---------|----------------|-------------|
| `updater.check` | raw manifest JSON | Parses the manifest, compares `manifest.version` to `state.current_version`, sets `state.update_available` |
| `updater.download` | platform key (e.g. `"macos-aarch64"`) | Stub; sets `state.downloaded = false` and `state.archive_path = null` |
| `updater.install` | none | Stub; sets `state.installed = false` and `state.staging_path = null` |

## State

- `state.current_version` — the running app's version (passed to `create`)
- `state.manifest_url` — URL the manifest would be fetched from in production
- `state.public_key` — optional 32-byte Ed25519 public key (decoded from base64) for signature verification
- `state.update_available` — `true` when the parsed manifest version is greater than the current version
- `state.manifest` — the most recently parsed `update_manifest.Manifest`
- `state.downloaded` / `state.installed` — flags left `false` until the stubs are wired up

## JS example

```javascript
import { invoke } from 'zero-native';
const manifestJson = '{"version":"1.0.0","notes":"major","platforms":{}}';
await invoke('updater.check', manifestJson);
const result = await invoke('updater.download', 'macos-aarch64');
```

## Notes

- `create(allocator, io, current_version, manifest_url, public_key_b64)` accepts an empty `public_key_b64` to skip signature verification.
- Production use requires wiring `http_client.request` into `updater.check` to fetch the manifest from `state.manifest_url`.
- The capability is registered as `Capability{ .kind = .custom, .name = "updater" }`.
- Version comparison is performed with `update_manifest.compareVersion`; a manifest at the same version reports `update_available = false` (no upgrade prompt).
- Signature verification is currently a stub; provide a base64-encoded Ed25519 public key to opt in.

## Lifecycle

```zig
const module = try plugin_updater.create(allocator, io, "1.0.0", "https://example.com/manifest.json", "");
try registry.startAll(runtime);
// ... dispatch updater.check, .download, .install ...
try registry.stopAll(runtime);
```

`stop` deinits the parsed manifest, frees the duplicated `current_version` and `manifest_url` slices, plus any archive/staging paths, and destroys the state struct. See [Updates](/updates) for the broader update workflow.

See [Extensions](/extensions) for the module lifecycle.
