# App Sandbox

macOS App Sandbox can be enabled by adding the `security.sandbox` block to the manifest. The packaging step generates a dynamic entitlements plist from the configuration and embeds it in the signed `.app` bundle.

```toml
[security.sandbox]
sandbox = true
network_client = true
files_user_selected_read = true
allow_jit = true
file_read = ["$APPDATA/**"]
```

`$APPDATA` is a placeholder the packager expands to the app's sandbox container path at build time.

## Generated plist

The example manifest produces this entitlements plist (the packager signs the bundle with this plist embedded):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0"><dict>
  <key>com.apple.security.app-sandbox</key><true/>
  <key>com.apple.security.network.client</key><true/>
  <key>com.apple.security.files.user-selected.read-only</key><true/>
  <key>com.apple.security.cs.allow-unsigned-executable-memory</key><true/>
  <key>com.apple.security.temporary-exception.files.absolute-path.read-only</key>
  <array><string>$APPDATA/**</string></array>
</dict></plist>
```

## Zig API

```zig
const sandbox = zero_native.security.sandbox.MacOSSandbox{
    .sandbox = true,
    .network_client = true,
    .allow_jit = true,
    .file_read = &.{ "$APPDATA/**" },
};
const entitlements = try sandbox.entitlements(allocator);
defer sandbox.freeEntitlements(allocator, entitlements);
const plist_xml = try sandbox.toPlist(allocator, entitlements);
defer allocator.free(plist_xml);
```
