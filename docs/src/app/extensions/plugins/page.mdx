# Bundled Plugins

zero-native ships with 16 reference plugins demonstrating common patterns. Each plugin lives in `src/extensions/plugin_<name>.zig` and registers through the same `ModuleRegistry` hook system described in the [Extensions](/extensions) page.

## Conventions

All plugins follow the same conventions:

- Commands are routed by `cmd.name` only. Any argument string lives in `cmd.payload`.
- Common shapes for payloads: `arg1|arg2|...` (pipe-separated), or a single key/value pair.
- Each plugin exposes `create(allocator, ...)` returning an `extensions.Module` and `command(ctx, runtime, cmd)`.
- Tests inspect plugin state directly because the extension `Command` has no return channel.

## Wiring a plugin

```zig
const extensions = @import("zero-native").extensions;
const clipboard_plugin = @import("zero-native/src/extensions/plugin_clipboard.zig");

const modules = [_]extensions.Module{
    try clipboard_plugin.create(allocator),
};

var runtime = try zero_native.Runtime.init(.{
    .platform = my_platform,
    .extensions = .{ .modules = &modules },
});
```

See [Extensions](/extensions) for the full module and hook API.
