# Plugin: Clipboard

Text clipboard with command dispatch. The plugin keeps an in-memory buffer as the authoritative fallback and attempts to sync reads and writes with the host OS native clipboard when platform utilities are available.

## Commands

| Command | Payload format | Description |
|---------|----------------|-------------|
| `clipboard.write_text` | `text` (whole payload) | Writes the payload to the in-memory buffer and tries to push it to the native OS clipboard; empty payload falls back to `default_text` (`"hello from test"`) |
| `clipboard.read_text` | none | Tries to read from the native OS clipboard; if that fails or returns nothing, duplicates the in-memory buffer into `state.last_read` for inspection |

## Native clipboard behavior

The plugin invokes external platform utilities via `std.process.Child`:

- **macOS**: `pbcopy` for write, `pbpaste` for read.
- **Linux**: `wl-copy` / `wl-paste` (Wayland) are preferred; `xclip -selection clipboard -in` / `xclip -selection clipboard -out` (X11) are used as a fallback.
- **Windows**: `clip` for write; `powershell.exe -Command Get-Clipboard` for read.

If the platform utility is missing, the display server is unreachable, or the child exits unsuccessfully, the plugin returns an error and the command hook falls back to the in-memory buffer. The in-memory buffer is therefore always safe to rely on.

## State

- `state.text` — most recent text written (or `null` initially)
- `state.last_read` — copy returned by the last `clipboard.read_text` call
- `state.allocator` — allocator used for the duplicated slices (freed in `stop`)

## JS example

```javascript
import { invoke } from 'zero-native';
await invoke('clipboard.write_text', 'hello world');
const result = await invoke('clipboard.read_text', '');
console.log(result);
```

## Notes

- The in-memory buffer is the source of truth when the native clipboard is unavailable or fails.
- The capability is registered as `Capability{ .kind = .clipboard }`; queries via `registry.hasCapability(.clipboard)` return `true` once the plugin is installed.
- Pair with [Builtin Commands](/bridge/builtin-commands) when you need a `window.zero.clipboard.*` helper for the WebView side.
- The `default_text` constant is exported for tests; production callers should always pass a non-empty payload.
- On Windows, `Get-Clipboard` may append a trailing newline to its console output; the plugin trims one trailing newline to preserve round-trip fidelity.

## Lifecycle

```zig
const module = try plugin_clipboard.create(allocator);
try registry.startAll(runtime);
// ... dispatch commands ...
try registry.stopAll(runtime);
```

`stop` frees the duplicated `text` and `last_read` slices, tears down the internal threaded IO instance used for spawning clipboard utilities, then destroys the state struct. Leaks will be flagged by `std.testing.allocator` if you use the testing allocator in tests.

See [Extensions](/extensions) for the module lifecycle.
