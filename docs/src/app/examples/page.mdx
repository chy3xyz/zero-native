The `hello` and `webview` examples are the most useful starting points for understanding the framework; the `next`, `react`, `svelte`, and `vue` examples show how to wire a Vite-based or Next-based frontend into the native shell; the `android` and `ios` examples show how to host the framework on mobile.

## Run an example

Each example is a self-contained Zig project. From the example's directory:

```bash
cd examples/hello
zig build run
```

The first run downloads frontend dependencies and builds the native shell. Subsequent runs are cached.

## Create your own

The fastest way to start a new project is the `init` command, which scaffolds a fresh `app.zon`, `build.zig`, `src/main.zig`, and (optionally) a frontend:

```bash
npx zero-native init my-app --frontend <vite|react|svelte|vue|next>
```

After scaffolding, `cd my-app && zig build run` builds and launches the app. `zig build dev` starts the frontend's dev server and points the WebView at it. See [CLI Reference](/cli) for the full option list and [Quick Start](/quick-start) for the end-to-end walkthrough.
